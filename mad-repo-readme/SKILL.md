# Repo README

## Instructions

- Keep the README short and practical: what the repo is, how to install/use, how to sync or contribute when relevant.
- Default language: **English** unless the user requests otherwise.
- Prefer concrete commands and links over narrative.

## Do not include layout

Do **not** document the repository file/directory layout (trees, folder listings, “Layout” / “Structure” sections).

The tree is already visible in Git hosting (GitHub/GitLab file browser) and locally — duplicating it in the README goes stale and adds no value.

Omit layout even if other README templates suggest including it. Only list paths when a specific command or config requires an exact path (e.g. install symlink target).
